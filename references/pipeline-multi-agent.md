# Multi-Agent E-commerce 流水线详解（主编调度手册）

> 本文档是 [SKILL.md](SKILL.md) 的完整流水线细节，供主编调度时参考。

---

## 流水线概览

```
客户发送消息
         ↓
主编接收消息，初筛判断
         ↓
   ┌─────┴─────┐
   ↓           ↓
简单场景    复杂场景
直接回答    启动流程
   ↓           ↓
   └─────┬─────┘
         ↓
   意图识别Agent分析
         ↓
   ┌─────┴─────┐
   ↓           ↓
情绪检测    识别意图
   ↓           ↓
   └─────┬─────┘
         ↓
   路由到专业Agent
   （订单/产品/投诉/售后/物流/优惠）
         ↓
主编组装最终回复
         ↓
客户收到解答
```

---

## 主编决策树

```
客户消息：[客户输入]

1. 是否包含敏感操作词（退款>200/欺诈/报警）？
   是 → ❌ 转人工，告知客户稍等
   否 → 继续判断

2. 是否是简单咨询（问候/单个问题/明确意图）？
   是 → 直接路由到对应Agent
   否 → 启动意图识别流程

3. 情绪是否激动（感叹号多/重复词/情绪词）？
   是 → 先执行安抚话术，再正常处理
   否 → 正常流程
```

## 主编调度5步法

### Step 1: 消息接收与初筛

主编收到客户消息后：

```markdown
## 初筛判断

**客户消息**：[消息内容]
**情绪检测**：平静/不满/激动
**敏感操作**：是/否
**意图识别**：模糊/明确

**判断结果**：
- ✅ 直接处理：[Agent名]
- ⚠️ 需要流程：[启动多Agent流程]
- ❌ 转人工：[原因]
```

### Step 2: 意图识别（如需要）

启动意图识别Agent：

```markdown
## 意图识别任务

请分析以下客户消息，识别客户意图：

**客户消息**：[消息内容]
**客户历史**：[如有]

**6大模块**：
1. 订单查询 - 查订单、发货、付款
2. 产品咨询 - 产品好不好、规格、对比
3. 投诉处理 - 投诉、差评、不满意
4. 售后处理 - 退货、换货、坏了
5. 物流查询 - 物流、快递、到哪了
6. 优惠咨询 - 优惠、折扣、优惠券

**输出**：意图分类报告（见agents.md）
```

### Step 3: 情绪检测

意图识别后，同步进行情绪检测：

```markdown
## 情绪检测任务

**客户消息**：[消息内容]

**情绪词扫描**：
- 🚨 激动词：[列表]
- 📊 不满词：[列表]
- ✅ 平静词：[列表]

**情绪状态**：平静/不满/激动

**应对策略**：
- 激动：先安抚3句再处理
- 不满：先道歉1句再处理
- 平静：直接处理
```

### Step 4: 路由到专业Agent

根据意图路由到对应Agent：

```markdown
## [Agent名] 任务

**客户意图**：订单查询
**情绪状态**：平静
**客户身份**：手机138****1234

**执行要求**：
请执行[skill_xxx.md]中的流程，返回回复内容。

**输出格式**：
- 回复客户的内容
- 如需补偿：补偿方案+金额
- 如需转人工：转人工原因
```

### Step 5: 组装最终回复

主编收集Agent输出后，组装最终回复：

```markdown
## 最终回复

**客户消息**：[原始消息]
**处理Agent**：[Agent名]
**情绪状态**：[状态]

**回复内容**：
[Agent输出的回复]

**附加操作**：
- [ ] 已发送补偿券
- [ ] 已创建工单
- [ ] 已通知物流

**客户满意度预估**：✅ 满意 / ⚠️ 需跟进
```

## 6大客服模块执行详解

### 订单查询Agent执行流程

```
客户：我的订单到哪了？
主编 → 意图识别：订单查询
     → 情绪检测：平静
     → 路由到订单查询Agent
订单Agent → 身份验证（手机后4位）
         → query_order查询
         → 状态解读（已发货）
         → 输出：物流信息+预计到达
主编 → 组装回复 → 客户
```

### 投诉处理Agent执行流程

```
客户：太差了！等了一周还没到！
主编 → 意图识别：物流投诉（物流延迟+情绪激动）
     → 情绪检测：激动 🚨
     → 路由到投诉处理Agent
投诉Agent → 安抚话术（3句）
         → 问题分类：物流延迟
         → 补偿标准：订单金额5%，最高50元
         → 执行补偿
         → 输出：道歉+补偿方案+预计物流
主编 → 组装回复 → 客户
```

### 售后Agent执行流程

```
客户：产品坏了，能退货吗？
主编 → 意图识别：售后处理
     → 情绪检测：平静
     → 路由到售后Agent
售后Agent → 订单验证（确认产品+订单）
         → 判断：7天内/15天内/超期
         → 流程：7天内→七天无理由
         → 输出：退货地址+流程+预计退款
主编 → 组装回复 → 客户
```

## 转人工标准

以下情况必须转人工：

| 情况 | 原因 | 操作 |
|------|------|------|
| 退款>200元 | 超出AI权限 | 创建工单，通知客服 |
| 涉及欺诈投诉 | 法律风险 | 创建工单，通知主管 |
| 情绪失控 | 需人工安抚 | 直接转接 |
| 复杂技术问题 | 超出知识库 | 创建工单 |
| 媒体/记者 | 舆情风险 | 通知公关部 |

## 执行示例

**场景1：简单查询**
```
客户：查一下我138****1234的订单

① 主编初筛：简单场景，直接路由
② 意图识别：订单查询
③ 情绪检测：平静
④ 订单Agent：query_order，返回状态"已发货"
⑤ 主编：组装回复，输出物流信息
```

**场景2：情绪投诉**
```
客户：太差了！东西等了一周还没到！

① 主编初筛：情绪激动+物流投诉
② 意图识别：物流延迟投诉
③ 情绪检测：激动（🚨）
④ 投诉Agent：安抚3句 + 物流延迟补偿5%
⑤ 主编：组装回复，道歉+补偿+物流更新
```

**场景3：转人工**
```
客户：我要投诉你们欺骗消费者！

① 主编初筛：欺诈投诉 🚨
② 意图识别：投诉处理
③ 情绪检测：激动
④ 判断：欺诈投诉 → 需转人工
⑤ 主编：转人工，告知客户稍等
⑥ 创建工单通知客服主管
```
