# 意图识别分类器

> 意图识别Agent使用此文档进行准确的意图分类。

---

## 意图分类体系

### 6大主意图

| 意图ID | 意图名称 | 优先级 | 说明 |
|--------|----------|--------|------|
| INTENT-001 | 订单查询 | P3 | 查订单状态、发货时间、付款问题 |
| INTENT-002 | 产品咨询 | P4 | 产品规格、使用方法、对比 |
| INTENT-003 | 投诉处理 | P1 | 投诉、差评、不满意、欺诈 |
| INTENT-004 | 售后处理 | P2 | 退货、换货、维修、三包 |
| INTENT-005 | 物流查询 | P2 | 查物流、快递进度、签收 |
| INTENT-006 | 优惠咨询 | P5 | 优惠、折扣、优惠券、活动 |

---

## 关键词匹配规则

### INTENT-001 订单查询
**触发词**：
- 查订单、我的订单、订单在哪
- 发货了吗、什么时候发货
- 订单状态、订单明细
- 付款了吗、没付款、支付

**排除词**：
- 取消订单 → INTENT-004 售后处理
- 修改地址 → 转人工

### INTENT-002 产品咨询
**触发词**：
- 产品好不好、产品质量
- 规格是什么、参数
- 这个能用吗、适合吗
- 和XX对比、哪个好

**排除词**：
- 价格 → INTENT-006 优惠咨询

### INTENT-003 投诉处理
**触发词**：
- 投诉、差评、太差了
- 不满意、骗人、欺骗
- 质量问题、破烂、损坏
- 举报、投诉到315

**排除词**：无（最高优先级）

### INTENT-004 售后处理
**触发词**：
- 退货、换货
- 坏了不能用、维修
- 三包、在保修期
- 退款（小额）

**排除词**：
- 退款>200元 → 转人工

### INTENT-005 物流查询
**触发词**：
- 物流、快递
- 到哪了、在哪
- 什么时候到、几天到
- 签收、没收到

**排除词**：
- 物流投诉 → INTENT-003 投诉处理

### INTENT-006 优惠咨询
**触发词**：
- 优惠、打折
- 优惠券、券
- 有活动吗、促销
- 便宜点、能减吗

**排除词**：无

## 意图分类决策树

```
客户消息
   ↓
扫描情绪词（投诉相关？）
   ↓ 是 → INTENT-003 投诉处理
   ↓ 否
扫描售后词（退货/换货/维修）？
   ↓ 是 → INTENT-004 售后处理
   ↓ 否
扫描物流词（快递/物流/到哪）？
   ↓ 是 → INTENT-005 物流查询
   ↓ 否
扫描订单词（订单/发货/付款）？
   ↓ 是 → INTENT-001 订单查询
   ↓ 否
扫描优惠词（优惠/折扣/券）？
   ↓ 是 → INTENT-006 优惠咨询
   ↓ 否
扫描产品词（产品/规格/对比）？
   ↓ 是 → INTENT-002 产品咨询
   ↓ 否
   → 综合咨询（主编判断）
```

## 多意图处理

### 优先级规则
当一个消息包含多个意图时：
1. **投诉类最高** → INTENT-003 优先处理
2. **售后类次之** → INTENT-004 第二处理
3. **其他按序** → 按识别顺序处理

### 示例
```
"东西坏了要退货，而且物流也太慢了！"

识别到：
- INTENT-004 售后处理（退货）
- INTENT-003 投诉处理（物流慢投诉）

优先级：INTENT-003 > INTENT-004

执行：先处理投诉（安抚+补偿），再处理售后（退货流程）
```

## 置信度评估

| 置信度 | 标准 | 处理方式 |
|--------|------|----------|
| 95%+ | 关键词明确匹配 | 直接路由 |
| 80-94% | 主要词匹配，略有歧义 | 路由+追问确认 |
| 60-79% | 部分词匹配 | 路由+主动确认 |
| <60% | 模糊/无匹配 | 主编判断 |
